## PERSONAL SECURITY HARDENING RECOMMENDATIONS

### High Priority (Good to implement)
1. Set explicit `PermitRootLogin no` in `/etc/ssh/sshd_config` (not just comment it out)
2. Add `ufw deny 22` when not needed (uncomment when using remote SSH)
3. Keep fail2ban active - it's already protecting SSH
4. Consider enabling fail2ban logging to review ban patterns

### Medium Priority (Monitor)
1. Regularly inspect `.vscode-server` and `.claude` directories for unexpected modifications
2. Monitor `/tmp/` contents for accumulation of old logs
3. Review `.codex` logs regularly for unexpected behavior
4. Consider restricting sudo access limits if not needed constantly

### Low Priority (Optional)
1. `harden.sh` allows sudo without password prompt for this user - consider adding command restrictions
2. Empty `authorized_keys` file is fine but could be removed to reduce attack surface (no keys will ever be accepted unless populated)
3. Consider logging all interaction with `/home/ubuntu/reports/` directory
